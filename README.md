# 🏹 Drona AI

**AI-Powered Adaptive Learning Management System** powered by **Groq LLM (Llama 3.3 70B)**.

A multi-agent LMS where every response — explanations, quizzes, evaluations, scheduling — is generated by AI. Built for college students studying ML, DSA, and Database Systems. Named after the legendary teacher Dronacharya, it acts as a hyper-intelligent, 24/7 personalized mentor that dynamically adapts its curriculum generation to the exact cognitive pace, strengths, and weaknesses of the individual student.

---

## ✨ Features

| Feature | How It Works |
|---------|-------------|
| **📖 AI Explanations** | Uses TF-IDF vector similarity search against verified collegiate PDFs to ground all AI explanations in academic truth, preventing hallucination. |
| **📝 Dynamic Quizzes** | Generates syllabus-aligned MCQs on-the-fly, adjusting vocabulary and conceptual depth based on the student's adaptive difficulty level. |
| **📊 Smart Evaluation** | Analyzes *why* the student's chosen answer was incorrect and writes a personalized micro-explanation for every single mistake. |
| **📅 Natural Scheduling** | Natural language temporal parsing that detects events, assignments, and deadlines purely from casual chat messages. |
| **📈 Progress Tracking** | Dedicated algorithmic dashboards track Topic Mastery for individual subjects, culminating in a 3-paragraph AI-generated future study plan. |
| **🔄 Swappable LLM** | Flexible backend to switch between Groq (Llama 3), OpenAI (GPT-4o), or Ollama deployments via simple environment variables. |

## 🎯 The Problem
The contemporary education system and legacy LMS platforms suffer dramatically from a **systemic one-size-fits-all approach**:
* **Static Assessment Bottlenecks:** Students receive the exact same static quizzes. Once memorized, the assessment loses pedagogical value.
* **Delayed Feedback Loops:** When students encounter roadblocks, they wait days for office hours or rely on generic external forums.
* **Superficial Academic Analytics:** Traditional grading ("75/100") lacks granularity. Students don't know *why* they failed specific micro-topics.
* **Fragmented Study Planning:** Students lack the executive function to automatically map raw exam data into an effective daily study plan.

## 💡 Unique Selling Propositions (USPs)

1. **Infinite Adaptive Generation:** Drona AI uses its *Quiz Agent* to synthetically generate infinite, unique variations of MCQs in real-time.
2. **Orchestrated Multi-Agent Architecture:** A specialized ecosystem where the *Router Agent* categorizes intent, the *Query Agent* executes RAG, and the *Evaluation Agent* strictly grades submissions.
3. **Hyper-Granular Micro-Analytics:** Breaks down broad subjects into micro-topics, tracking mastery percentages for each to intelligently target weaknesses.
4. **Seamless Conversational Infrastructure:** Students simply type *"I have an ML quiz next Friday at 10 AM,"* and the *Scheduler Agent* handles the rest.

## 🏗️ Architecture & Core Flow

```text
[ Client Layer (Next.js 15 App Router / Glassmorphism UI) ]
         │ (HTTP POST with JSON Payloads)
         ▼
[ API Orchestration Layer (/api/chat & /api/quiz) ]
         │
         ▼
[ Router Agent (LLM classification engine) ] ───▶ Determines execution path
         │
         ├─▶ [ Query Agent ] ──▶ [ TF-IDF Vector Search ] ──▶ [ PDF Knowledge Base ]
         │                  (Retrieval Augmented Generation)
         │
         ├─▶ [ Quiz Agent ] ──▶ Automatically scales difficulty via context
         │
         ├─▶ [ Scheduler Agent ] ──▶ Temporal Datetime Parsing
         │
         └─▶ [ Evaluation Agent ] ──▶ Generates deep-dive analytical feedback
         │
         ▼
[ Provider-Agnostic LLM Engine (Groq Llama 3.3 70B / OpenAI / Local Ollama) ]
         │
         ▼
[ Data Persistence Layer (SQLite Database managed via Prisma ORM) ]
```

### 🎓 How Each Flow Works

* **Explanation Flow:** User asks → Router classifies intent → Query Agent searches RAG knowledge base → LLM generates formatted explanation with sources.
* **Quiz Flow:** User requests quiz → Router classifies intent → Quiz Agent generates MCQs → User answers → Quiz Submit API grades → Evaluation Agent analyzes performance per question.
* **Schedule Flow:** User states exam date → Router classifies intent → Scheduler Agent parses date/time → Saves to SQLite database.

## 🔑 Key File: `src/lib/llm.ts`

The LLM client is the heart of the system. **Swap LLM provider** by changing 3 env vars:

| Provider | `LLM_BASE_URL` | `LLM_API_KEY` | `LLM_MODEL` |
|----------|----------------|---------------|-------------|
| **Groq** | `https://api.groq.com/openai/v1` | `gsk_xxx` | `llama-3.3-70b-versatile` |
| OpenAI | `https://api.openai.com/v1` | `sk-xxx` | `gpt-4o-mini` |
| Ollama | `http://localhost:11434/v1` | `ollama` | `llama3` |
