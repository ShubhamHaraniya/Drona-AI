# 🏹 Drona AI

**AI-Powered Adaptive Learning Management System** powered by **Groq LLM (Llama 3.3 70B)**.

A multi-agent LMS where every response — explanations, quizzes, evaluations, scheduling — is generated by AI. Built for college students studying ML, DSA, and Database Systems.

---

## ✨ Features

| Feature | How It Works |
|---------|-------------|
| **📖 AI Explanations** | LLM generates detailed explanations using RAG (Retrieval-Augmented Generation) from the knowledge base |
| **📝 Dynamic Quizzes** | LLM generates unique MCQ questions on any topic, any difficulty |
| **📊 Smart Evaluation** | LLM analyzes your quiz answers — per-question ✅/❌ review with explanations |
| **📅 Natural Scheduling** | LLM understands any date/time format — "ML exam 24 march 4 pm" just works |
| **📈 Progress Tracking** | Adaptive difficulty, topic mastery, streaks, confidence calibration |
| **🔄 Swappable LLM** | Change 3 env vars to switch from Groq → OpenAI → Ollama |

## 🏗️ Architecture

```
User Message
    ↓
┌─────────────────────────────┐
│  Router Agent (LLM)         │  ← Classifies intent via LLM
│  Returns: intent, subject,  │
│           topic, params     │
└─────────┬───────────────────┘
          ↓
    ┌─────┴─────┬─────────┬──────────┐
    ↓           ↓         ↓          ↓
┌─────────┐ ┌────────┐ ┌────────┐ ┌──────────┐
│ Query   │ │ Quiz   │ │ Eval   │ │ Schedule │
│ Agent   │ │ Agent  │ │ Agent  │ │ Agent    │
│ (LLM+   │ │ (LLM)  │ │ (LLM)  │ │ (LLM)    │
│  RAG)   │ │        │ │        │ │          │
└─────────┘ └────────┘ └────────┘ └──────────┘
```

All agents call the **LLM Client** (`src/lib/llm.ts`) which uses the OpenAI-compatible API format.

## 🔑 Key File: `src/lib/llm.ts`

The LLM client is the heart of the system. It provides 3 functions:

```typescript
chatCompletion(messages, options)  // Full chat completion
ask(systemPrompt, userMessage)     // Quick string response
askJSON<T>(systemPrompt, message)  // Parsed JSON response (typed)
```

**Swap LLM provider** by changing 3 env vars:

| Provider | `LLM_BASE_URL` | `LLM_API_KEY` | `LLM_MODEL` |
|----------|----------------|---------------|-------------|
| **Groq** | `https://api.groq.com/openai/v1` | `gsk_xxx` | `llama-3.3-70b-versatile` |
| OpenAI | `https://api.openai.com/v1` | `sk-xxx` | `gpt-4o-mini` |
| Ollama | `http://localhost:11434/v1` | `ollama` | `llama3` |

## 🎓 How Each Flow Works

### Explanation Flow
1. User: "Explain backpropagation"
2. **Router Agent** → classifies as `Explanation`, subject: `ML`
3. **Query Agent** → searches RAG knowledge base → sends context + question to LLM
4. LLM generates detailed, formatted explanation with sources

### Quiz Flow
1. User: "Quiz me on 5 ML questions"
2. **Router Agent** → classifies as `QuizRequest`, params: `{ count: 5 }`
3. **Quiz Agent** → LLM generates 5 MCQs with correct answers + explanations
4. User answers questions → submits
5. **Quiz Submit API** → grades against stored answer key
6. **Evaluation Agent** → LLM analyzes performance, shows ✅/❌ per question

### Schedule Flow
1. User: "I have DBMS quiz on 24 march 4 pm"
2. **Router Agent** → classifies as `Scheduling`
3. **Scheduler Agent** → LLM parses date/time → saves to database

## 🛠️ Tech Stack

- **Framework:** Next.js 15 (App Router)
- **LLM:** Groq (Llama 3.3 70B) via OpenAI-compatible API
- **Database:** SQLite via Prisma ORM
- **Auth:** NextAuth.js (credentials)
- **RAG:** TF-IDF vector store with cosine similarity
- **Language:** TypeScript
